# Milestone v1.0: blueCode MVP

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 1-5
**Total Plans:** 17 (16 autonomous + 1 human-gated UAT)

## Overview

blueCode is a strong-typed F# agent loop that drives locally-served Qwen 32B/72B on a Mac. The build followed a single hard constraint: domain types compile before everything else, the LLM client and tool executor must both exist before the agent loop can run, and the security layer arrives before any live Qwen shell command executes. Five phases delivered a working daily-driver that replaces the Python `claw-code-agent`.

v1.0 is complete: all 5 phases shipped, 39 v1 requirements Complete, 208 tests pass, Qwen 32B + 72B both chat-correct end-to-end, and the daily-driver cutover (`~/projs/claw-code-agent/` → `-retired/`) is done. UAT run succeeded against live Qwen with a real coding task ("List files in src" → 2 steps, exit 0).

### Key Decisions

From `.planning/STATE.md` Accumulated Decisions (highlights):

**Phase 1 / Foundation:**
- `task {}` exclusively in Core; `async {}` banned by CI grep script in Core.fsproj
- NuGet packages locked at project init: FSharp.SystemTextJson 1.4.36, FsToolkit.ErrorHandling 5.2.0, JsonSchema.Net 9.2.0, Spectre.Console 0.55.2, Argu 6.2.5, Serilog 4.3.1
- F# compile order enforced — Domain.fs first, adapters last
- `ToolResult` DU shape ships in Phase 1 (FND-02 expanded) to pair with `Tool` DU for exhaustive-match proof; full semantic contract in Phase 3
- `classifyIntent` keyword priority: Debug > Design > Analysis > Implementation > General (first-match-wins)
- Ports-and-adapters invariant: BlueCode.Core has zero references to Serilog, Spectre.Console, Argu

**Phase 2 / LLM Client:**
- FSharp.SystemTextJson 1.4.36 API: `JsonSerializerOptions() + opts.Converters.Add(JsonFSharpConverter())` (no `ToJsonSerializerOptions()` method)
- `JsonFSharpOptions.Default().WithUnionUnwrapFieldlessTags(true)` → bare-string DU encoding (`"System"` not `{"Case":"System"}`)
- `TaskCanceledException` disambiguation: `ex.CancellationToken = ct` → UserCancelled, else → LlmUnreachable (timeout)

**Phase 3 / Tool Executor:**
- `/bin/bash -c` (not `/bin/sh`) — BashSecurity validators assume bash semantics
- Process.Start wrapped in Ok/Error instead of try/finally to avoid F# task CE type constraint errors with nested try/finally inside match branches
- Concurrent stdout/stderr read via F# 10 `let!/and!` — sequential read deadlocks when stderr buffer fills (dotnet/runtime #98347)
- Python `bash_security.py` has 22 validate_* functions (not 18 as plan spec said) — all 22 ported
- Fork bomb `:(){ :|:& };:` NOT blocked (matches Python behavior — no destructive pattern covers it)

**Phase 4 / Agent Loop:**
- `AgentLoop.fs` lives in BlueCode.Core (pure) — no Serilog/Spectre/Cli references
- `dispatchTool` inlined in AgentLoop.fs — ToolRegistry.fs stays empty stub; no premature abstraction
- System prompt lives in `AgentConfig` record (not inline constant) — injectable for tests
- `onStep` callback threaded through runSession/runLoop — enables CLI-layer JSONL writer for crash-safe per-step logging
- `Step.Thought = "[not captured in v1]"` placeholder — capturing real thought deferred to v1.1+ (requires `ILlmClient.CompleteAsync` to return `Thought * LlmOutput`)
- JSONL uses stdlib `StreamWriter` with `AutoFlush=true` (not Serilog.Sinks.File) — deterministic flush semantics for crash-safety
- `standardErrorFromLevel = Nullable LogEventLevel.Verbose` routes ALL Serilog events to stderr (OBS-02 separation from Spectre.Console on stdout)

**Phase 5 / CLI Polish:**
- `CliArgs.fs` extracted to `src/BlueCode.Cli/` (not inline in Program.fs) — F# EntryPoint modules can't be `open`'d by test projects
- `AgentConfig.ForcedModel: Model option` is the **only** Core change in Phase 5 — every other Phase 5 change landed in BlueCode.Cli
- `LoggingLevelSwitch` is module-level `let` in Logging.fs; `ControlledBy(levelSwitch)` replaces `MinimumLevel.Debug()` for runtime flip on `--trace`
- `Log.Debug` always called in `onStep` — Serilog suppresses before formatting when switch is at Information; zero-cost gate
- `withSpinner` extended with CTS + background Task.Delay(500) ticker for live elapsed-second label (CLI-05)
- `printfn` (Console.Out) used for 80% WARNING over `AnsiConsole.MarkupLine` — AnsiConsole bypasses `Console.SetOut` in tests
- Integer-only 80% check: `totalChars*5 >= maxModelLen*16` avoids float
- Fantomas 7.0.5 installed as local dotnet tool; 35 files reformatted; format committed as isolated `style(05-03)` commit
- RouterTests.fs `rootTests` assembles all test lists explicitly — new test modules must be added there (Expecto `[<Tests>]` auto-discovery not used)
- `testSequenced` wraps ReplTests testList — Expecto runs testList in parallel by default; `Console.SetOut` is global state

**UAT / 05-04 findings:**
- `Router.modelToName` returns loaded model absolute path (not HF id) — mlx_lm.server resolves unknown ids against HuggingFace Hub (404). Unblocked via fix commit `5ab5a95`. v1.1 OBS-03 dynamic `/v1/models` lookup replaces this hardcode.
- mlx_lm.server 0.31.3 does NOT read external `chat_template.jinja` — only `tokenizer_config.json.chat_template` field. Instruct models ship with it inline; Base models don't.
- Initially-downloaded 32B (`mlx-community/qwen2.5-32b-mlx`) was Base Coder (FIM-only) — **not Instruct**. Detected via missing `special_tokens_map.json` + `added_tokens.json`. Re-download to `Qwen2.5-Coder-32B-Instruct` unlocked 32B chat post-milestone.
- `Log.Debug` in `QwenHttpClient.CompleteAsync` for POST body + response — added during UAT diagnostics, retained as feature (commit `ea18cf7`).
- Spinner label `"Thinking... [32B]"` parsed `[32B]` as Spectre color tag — escaped to `[[32B]]` (literal bracket escape).

- `Step.Thought` is literal `"[not captured in v1]"` — verbose mode displays the placeholder. Capturing real LLM thought requires changing `ILlmClient.CompleteAsync` return type — intentionally deferred to prevent Phase 4 scope creep.
- `Router.modelToName` hardcodes absolute local path `/Users/ohama/llm-system/models/qwen{32,72}b` — loses portability. Intended solution is v1.1 OBS-03 dynamic query (see above).
- Cross-turn memory in multi-turn REPL is explicitly POST-v1 — each turn is an independent `runSession` call; context accumulator is intra-turn only.
- 32B cold-start probe timing: `bootstrapAsync` waits on localhost:8000 `/v1/models` even when user targets 72B; causes transient WARN on cold Mac. Benign but noisy.
