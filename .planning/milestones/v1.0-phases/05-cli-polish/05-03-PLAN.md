    Position: after the existing `withSpinner` / `create` section, as a separate public function at module scope. Import `System.Text.Json.JsonDocument` (already imported) and `Serilog.Log` (if not already). Use the same `httpClient` module singleton.

    ```fsharp
    // ── /v1/models probe (OBS-03) ───────────────────────────────────────────────

    /// GET http://127.0.0.1:8000/v1/models and extract data[0].max_model_len.
    /// Returns the integer value on success; falls back to 8192 (conservative
    /// floor per research § Pattern 5) on any error: HTTP failure, JSON parse
    /// failure, missing/null field, non-positive value.
    ///
    /// Fallback path logs a Warning to Serilog (stderr). This is intentional —
    /// the probe failing is not a fatal error; it means the warning threshold
    /// is based on 8192 rather than the real limit.
    ///
    /// PUBLIC: called from CompositionRoot.bootstrapAsync.
    let getMaxModelLenAsync (ct: System.Threading.CancellationToken) : Task<int> =
        task {
            let fallback = 8192
            try
                use! resp = httpClient.GetAsync("http://127.0.0.1:8000/v1/models", ct)
                if not resp.IsSuccessStatusCode then
                    Serilog.Log.Warning(
                        "GET /v1/models returned {Status}; using fallback {Fallback}",
                        int resp.StatusCode, fallback)
                    return fallback
                else
                    let! json = resp.Content.ReadAsStringAsync(ct)
                    try
                        use doc = JsonDocument.Parse(json)
                        let root = doc.RootElement
                        match root.TryGetProperty("data") with
                        | true, data when data.ValueKind = JsonValueKind.Array && data.GetArrayLength() > 0 ->
                            let model0 = data.[0]
                            match model0.TryGetProperty("max_model_len") with
                            | true, el when el.ValueKind = JsonValueKind.Number ->
                                let ok, v = el.TryGetInt64()
                                if ok && v > 0L then return int v
                                else
                                    Serilog.Log.Warning(
                                        "max_model_len not a positive int; using fallback {Fallback}", fallback)
                                    return fallback
                            | _ ->
                                Serilog.Log.Warning(
                                    "max_model_len missing or non-numeric in /v1/models data[0]; using fallback {Fallback}",
                                    fallback)
                                return fallback
                        | _ ->
                            Serilog.Log.Warning(
                                "data array empty or missing in /v1/models response; using fallback {Fallback}",
                                fallback)
                            return fallback
                    with parseEx ->
                        Serilog.Log.Warning(
                            parseEx, "Failed to parse /v1/models response; using fallback {Fallback}", fallback)
                        return fallback
            with ex ->
                Serilog.Log.Warning(
                    ex, "GET /v1/models failed; using fallback {Fallback}", fallback)
                return fallback
        }
    ```

    Step A — Add a pure helper function to Repl.fs (module-level, above runSingleTurn) that decides whether the 80% warning should fire:

    ```fsharp
    /// Determine whether a context-window warning should fire.
    /// totalChars: accumulated character count of messages sent to LLM so far in this turn.
    /// maxModelLen: resolved max_model_len from /v1/models (in tokens).
    /// alreadyWarned: whether the warning has already been shown in this turn.
    ///
    /// Heuristic: totalTokens ≈ totalChars / 4 (research § Pattern 5,
    /// "Don't Hand-Roll"). Fire when totalChars >= maxModelLen * 4 * 0.80,
    /// which simplifies to totalChars * 5 >= maxModelLen * 16
    /// (integer-only, no floating-point).
    ///
    /// PUBLIC for testability.
    let shouldWarnContextWindow
        (totalChars: int)
        (maxModelLen: int)
        (alreadyWarned: bool)
        : bool
        =
        if alreadyWarned then false
        else
            // 80% of (maxModelLen * 4 chars) = maxModelLen * 16 / 5 chars
            // Equivalent: totalChars * 5 >= maxModelLen * 16
            int64 totalChars * 5L >= int64 maxModelLen * 16L
    ```

    Step B — Wire the accumulator into `runSingleTurn`. Per Open Question 3, scope is INTRA-TURN: a new mutable accumulator is created for each runSingleTurn call and reset when the function returns. In multi-turn mode, each new turn gets a fresh counter.

    Modify `onStep` in runSingleTurn:
    ```fsharp
    let mutable totalChars = 0
    let mutable warnedThisTurn = false

    let onStep (step: Step) =
        components.JsonlSink.WriteStep step
        printfn "%s" (renderStep renderMode step)
        // accumulate char count of action + result representations.
        // buildMessages in AgentLoop composes these per-turn; we approximate
        // by summing the step's visible content.
        let actionRepr = sprintf "%A" step.Action
        let resultRepr = sprintf "%A" step.ToolResult
        totalChars <- totalChars + actionRepr.Length + resultRepr.Length
        if shouldWarnContextWindow totalChars components.MaxModelLen warnedThisTurn then
            Spectre.Console.AnsiConsole.MarkupLine(
                sprintf "[yellow]WARNING: context at 80%% of model limit (%d chars, max_model_len=%d tokens ≈ %d chars). Next step may truncate.[/]"
                    totalChars components.MaxModelLen (components.MaxModelLen * 4))
            warnedThisTurn <- true
        // trace-mode Log.Debug (from Plan 05-02) stays unchanged
        Log.Debug(
            "Step {Number}: action={Action} elapsed_ms={DurationMs} input={Input} output={Output}",
            step.StepNumber, step.Action, step.DurationMs, actionRepr, resultRepr)
    ```

    NOTE: `MarkupLine` requires opening `Spectre.Console` at the top of Repl.fs. Add `open Spectre.Console` to the imports if not present.

    ```fsharp
    module BlueCode.Tests.ContextWarningTests

    open Expecto
    open BlueCode.Cli.Repl

    let tests =
        testList "Repl.shouldWarnContextWindow" [

            testCase "below 80% threshold → no warning" <| fun _ ->
                // maxModelLen = 8192, max chars ≈ 32768, 80% ≈ 26214
                Expect.isFalse (shouldWarnContextWindow 10000 8192 false)
                    "10K chars of 32K budget should not warn"

            testCase "exactly at 80% → warn" <| fun _ ->
                // 80% of 8192*4 = 26214.4 ; use 26215 to clear integer boundary
                Expect.isTrue (shouldWarnContextWindow 26215 8192 false)
                    "26215 chars of 32K budget should warn"

            testCase "above 80% → warn" <| fun _ ->
                Expect.isTrue (shouldWarnContextWindow 30000 8192 false) ""

            testCase "already warned this turn → suppressed" <| fun _ ->
                Expect.isFalse (shouldWarnContextWindow 30000 8192 true)
                    "Subsequent triggers in same turn are suppressed"

            testCase "larger model (32768 max_model_len) scales threshold" <| fun _ ->
                // 80% of 32768*4 = 104857.6
                Expect.isFalse (shouldWarnContextWindow 100000 32768 false) ""
                Expect.isTrue  (shouldWarnContextWindow 110000 32768 false) ""

            testCase "zero chars never warns" <| fun _ ->
                Expect.isFalse (shouldWarnContextWindow 0 8192 false) ""

            testCase "small model (2048) fires warning earlier" <| fun _ ->
                // 80% of 2048*4 = 6553.6
                Expect.isFalse (shouldWarnContextWindow 6000 2048 false) ""
                Expect.isTrue  (shouldWarnContextWindow 7000 2048 false) ""
        ]
    ```
