## v1 Requirements (all Complete)

v1 Minimal scope. 본인 `localLLM/` 설계 노트 + 연구 SUMMARY.md의 Phase 1 must-haves 기반.

### Foundation

- [x] **FND-01**: F# / .NET 10 솔루션이 `dotnet run`으로 빌드·실행된다 (BlueCode.Core + BlueCode.Cli 2-project)
- [x] **FND-02**: 핵심 도메인 타입이 Discriminated Union으로 정의된다 (`AgentState`, `Intent`, `Model`, `Tool`, `LlmOutput`, `AgentError`, `Step`, `ToolResult`). `ToolResult`의 DU shape (Success | Failure | SecurityDenied | PathEscapeBlocked | Timeout)은 Phase 1에서 Phase 3의 TOOL-07보다 먼저 정의된다.
- [x] **FND-03**: `task {}` CE 일관 사용 정책이 적용된다 (async {} 금지)
- [x] **FND-04**: `FsToolkit.ErrorHandling`의 `taskResult {}` CE로 에이전트 루프 에러 흐름이 표현된다

### LLM Client

- [x] **LLM-01**: OpenAI-compatible 엔드포인트(`localhost:8000`/`localhost:8001`)로 chat completion POST 가능
- [x] **LLM-02**: 응답 JSON을 추출 파이프라인(bare → brace-nest → fence-strip → ParseFailure)으로 파싱
- [x] **LLM-03**: `JsonSchema.Net`으로 LLM 출력 `{thought, action, input}` 스키마를 런타임 검증
- [x] **LLM-04**: `FSharp.SystemTextJson`의 `JsonFSharpConverter`가 옵션에 등록되고 DU 직렬화가 정상 동작
- [x] **LLM-05**: 32B는 temperature=0.2, 72B는 temperature=0.4로 하드코딩 (사용자 노출 금지)
- [x] **LLM-06**: HTTP 실패는 `AgentError.LlmUnreachable`로 매핑됨 (예외 누수 없음)

### Model Routing

- [x] **ROU-01**: `classifyIntent: string -> Intent` 순함수
- [x] **ROU-02**: `intentToModel: Intent -> Model` 순함수
- [x] **ROU-03**: `Model`을 엔드포인트로 변환해 LLM 클라이언트 호출
- [x] **ROU-04**: 사용자가 `--model 72b` / `--model 32b` 플래그로 라우팅을 강제 오버라이드 가능

### Tools

- [x] **TOOL-01**: `read_file` — 경로 + 옵션 line range
- [x] **TOOL-02**: `write_file` — 프로젝트 루트 밖 경로 거부
- [x] **TOOL-03**: `list_dir` — 비재귀 기본, depth-limit 옵션
- [x] **TOOL-04**: `run_shell` — 30s 타임아웃, stdout 100KB cap, stderr 10KB cap
- [x] **TOOL-05**: `run_shell` 보안 검증기 체인 — `bash_security.py` 22개 validator 포팅
- [x] **TOOL-06**: 모든 툴 출력 2000자 절단 (context overflow 방지)
- [x] **TOOL-07**: `ToolResult` DU semantic contract — 5 cases 컴파일 강제 매치

### Agent Loop

- [x] **LOOP-01**: 단일 turn prompt → LLM → tool → observe 최대 5회
- [x] **LOOP-02**: `MaxLoopsExceeded`는 `AgentError` 케이스
- [x] **LOOP-03**: 한 step당 한 툴, chaining 금지
- [x] **LOOP-04**: 동일 `(action, input_hash)` 3회 → 루프 가드 차단
- [x] **LOOP-05**: JSON 파싱 실패 2회 재시도 후 `InvalidJsonOutput`
- [x] **LOOP-06**: 불변 ring buffer 최근 N=3 step
- [x] **LOOP-07**: Ctrl+C graceful 종료, stack trace 없음

### CLI

- [x] **CLI-01**: 단일 호출 모드 `blueCode "<prompt>"`
- [x] **CLI-02**: 다중 turn REPL — `/exit` 또는 Ctrl+D 종료
- [x] **CLI-03**: `--verbose` 플래그 — thought/action/input/output/status 전체 출력
- [x] **CLI-04**: Compact 모드 step당 한 줄 요약
- [x] **CLI-05**: Spectre.Console spinner + 경과 시간 표시
- [x] **CLI-06**: Argu 기반 인자 파싱, `--help` 자동 생성
- [x] **CLI-07**: `--trace` — Serilog Debug JSON to stderr, `--verbose`와 독립

### Observability

- [x] **OBS-01**: JSONL 형식 `~/.bluecode/session_<timestamp>.jsonl`
- [x] **OBS-02**: Serilog 구조화 로그 stderr 출력
- [x] **OBS-03**: 시작 시 `/v1/models` max_model_len 조회, 80% 도달 시 경고
- [x] **OBS-04**: Step 타이밍 (startedAt/endedAt/durationMs)

## Milestone Summary

**Shipped:** 39 of 39 v1 requirements ✓

**Adjusted during implementation:**
- **OBS-04** (post-Phase 1 addition, 2026-04-22) — Step record timing fields. Added explicitly as additive extension; Phase 1's Step record declared with forward-compat intent.
- **CLI-07** (post-Phase 1 addition, 2026-04-22) — `--trace` flag. Added to scope during Phase 1 review.
- **TOOL-07** — split into two delivery points (DU shape in Phase 1 as part of FND-02; semantic contract in Phase 3). Counted as one requirement with two rows.
- **Router.modelToName** (post-milestone hotfix during 05-04 UAT, not a requirement change) — LLM-01's implementation hardcoded HF-style names. Corrected to absolute model paths to match `mlx_lm.server` resolution (commit `5ab5a95`). The requirement itself was unchanged.

**Dropped:** None. All 39 requirements delivered.

**Out of Scope (unchanged from v1 definition):** MCP, LSP, Plugin/hook, GUI, Permissions UI, Remote/worktree/team runtime, Windows/Linux, AOT, Claude prompt reuse, zero-dep constraint, multi-tool chaining, >5 step chains, LLM-based compaction, vision, background daemon, cost tracking, notebook editing, voice, telemetry.

**v1.1 carry-forward items** (recorded in STATE.md, not requirement scope changes):
- OBS-03 dynamic `/v1/models` model id query (replaces `Router.modelToName` absolute-path hardcode)
- Decouple 32B bootstrap probe so `--model 72b` mode doesn't trigger 32B cold-start WARN
- (Closed 2026-04-23 post-milestone) 32B Instruct re-download — documented in `documentation/qwen32b-base-to-instruct.md`
