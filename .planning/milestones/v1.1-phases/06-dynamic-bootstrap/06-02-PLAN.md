<objective>
Make bootstrap emit zero network I/O by deleting `bootstrapAsync` and the now-unused
`getMaxModelLenAsync` helper, collapsing startup onto the sync `bootstrap` function that
already produces the same `AppComponents` shape.

Purpose: satisfies REF-02 (lazy probe — bootstrap returns without touching the network).
The lazy-cache machinery needed for this plan is installed by plan 06-01; this plan just
removes the eager probe and its call site. SC-2 (no port-8000 WARN when `--model 72b` is
used with 8000 down) follows directly: since 06-01's per-port lazy probe only fires the
8000 probe on the first 32B `CompleteAsync`, using `--model 72b` never touches 8000.

Output:
- `CompositionRoot.fs` with only `bootstrap` remaining (no `bootstrapAsync`)
- `Program.fs` calling sync `bootstrap` — no `.GetAwaiter().GetResult()` on startup
- `QwenHttpClient.fs` with `getMaxModelLenAsync` deleted (only caller was `bootstrapAsync`)
- `ModelsProbeTests.fs` updated: the `getMaxModelLenAsync against closed port` test case
  rewritten to target `probeModelInfoAsync` instead (fallback path coverage preserved)
</objective>

       inside the `maxModelLenTests` sub-list (from plan 06-01 rename). It is the last case

    2. Replace the entire `testCase ... <| fun _ -> ...` block with a new case that uses
       `probeModelInfoAsync` against a deliberately-closed port:

       ```fsharp
       testCase "probeModelInfoAsync against closed port -> ModelId='' MaxModelLen=8192 fallback"
       <| fun _ ->
           // Port 64321 is highly unlikely to be listening; the connection will be refused,
           // triggering the catch path in probeModelInfoAsync which returns the fallback
           // record. This exercises the same WARN-and-fallback semantics that v1.0's
           // getMaxModelLenAsync had, now applied to the combined ModelInfo probe.
           let result =
               probeModelInfoAsync "http://127.0.0.1:64321" System.Threading.CancellationToken.None
               |> fun t -> t.GetAwaiter().GetResult()

           Expect.equal result.ModelId "" "closed-port fallback ModelId must be empty string"
           Expect.equal result.MaxModelLen 8192 "closed-port fallback MaxModelLen must be 8192 floor"
       ```

       Note the URL change: instead of hitting localhost:8000 (which might be live and
       return a real value, making the assertion flaky), we use port 64321 explicitly.
       This gives deterministic fallback-path coverage without depending on whether
       `mlx_lm.server` is running.

    3. Leave all other test cases (the 7 `tryParseMaxModelLen` cases + all 8 `tryParseModelId`
       cases from plan 06-01) untouched.

    **Pitfalls:**
    - Do NOT leave the old test referencing `getMaxModelLenAsync` — it must be replaced
      atomically with the new one (compile will fail otherwise).
    - Using port 64321: the connection refusal is near-instant on localhost, so this test
      runs fast. Do NOT add a timeout or CancellationToken configuration — the default
      httpClient singleton Timeout (180s) is not relevant for refused connections.
    - Do NOT assert on log output — `Serilog.Log.Warning` goes to stderr and we are not
      wiring a test sink. Structural Expect on the returned record is sufficient.
    - This is the ONLY test change needed. `RouterTests.rootTests` still references
